# 카페 앱 (Brew & Bean) — 발표 자료 정리

> PPT 제작용 원본 콘텐츠. 아래 섹션 순서대로 슬라이드 구성하면 됩니다.

---

## 1. 기획의도

- 카페에서 줄 서서 기다리는 시간이 아까워서, **미리 메뉴를 담고 주문해두면 매장에서 바로 픽업만 하면 되는 앱**을 만들어봤어요.
- 화면만 예쁘게 만드는 걸로 끝내지 않고, **데이터를 어떻게 저장할지 설계하는 것부터 화면 만들고 실제로 돌아가게 하는 것까지** 전체 과정을 한 번 다 해보는 게 목표였어요.
- 실제 결제(카드 승인 등)까지는 범위에서 빼고, "주문 접수 → 준비 중 → 픽업 완료"라는 흐름에만 집중했어요.
- 사용하는 사람은 두 종류예요: 메뉴 보고 담고 주문하는 **손님**, 그리고 메뉴/이벤트/주문을 관리하는 **사장님(관리자)**.

---

## 2. 기술 스택

| 영역 | 사용한 것 | 한 줄 설명 |
| --- | --- | --- |
| 화면 만들기 | HTML / CSS / 자바스크립트 | 별도 프로그램(프레임워크) 없이 기본 웹 기술만으로 만들었어요 |
| 데이터 저장 | 브라우저 저장소 | 서버 없이 내 브라우저 안에 데이터를 저장해두는 방식(추후 진짜 서버용 DB로 옮길 예정) |
| 로그인 | 직접 만든 간단한 로그인 기능 | 손님용 로그인이랑 관리자용 로그인을 완전히 따로 만들었어요 |
| 기능 테스트 | 자동 테스트 코드 | 버튼 하나 바꿀 때마다 눈으로 다 확인 안 해도 되게 테스트 23개를 만들어둠 |
| 화면 동작 확인 | 자동화 브라우저 테스트 | 실제 화면을 띄워서 클릭하고 입력해보는 걸 자동으로 반복 실행해서 확인 |
| 실행 환경 | 정적 파일 서버 | 만든 파일들을 브라우저에서 열어볼 수 있게 띄워주는 가벼운 프로그램 |
| 버전 관리 | 깃(Git) / 깃허브 | 작업한 내용을 기록으로 남기고 되돌릴 수 있게 관리 |
| 데이터 구조 설계 | ERD 다이어그램, 스키마 문서 | 나중에 진짜 데이터베이스로 옮길 걸 대비해서 표 형태로 미리 설계해둠 |
| 이미지 처리 | 이미지 축소/압축 | 사진을 그대로 저장하면 용량이 너무 커서 미리 줄여서 저장하게 함 |

**이렇게 만든 이유**: 서버나 진짜 데이터베이스 없이도 먼저 "실제로 써지는 느낌"까지 완성해보고 싶었어요. 화면이랑 흐름이 다 자리 잡은 다음에 진짜 서버로 옮기는 게 더 수월하다고 생각했거든요.

## 3. 주요 기능 (시연 포인트)

### 손님이 보는 화면
- **홈 화면**: 자동으로 넘어가는 배너 이미지, 오늘의 추천 메뉴, 공지사항, 이벤트 소개
- **메뉴 둘러보기**: 카테고리별로 보기, 검색, 정렬/가격 필터, 즐겨찾기, 최근 본 메뉴
- **메뉴 옵션**: 아이스/핫, 사이즈(레귤러/라지, 추가금액) 골라서 담기 → 가격이 바로바로 바뀜
- **장바구니**: 옵션 다르면 따로 담기, 디저트+음료 같이 담으면 자동 할인, 예상 준비 시간 안내
- **주문하기**: 받는 사람 정보 입력 → 주문 완료 → 준비 시간 실시간으로 카운트다운
- **주문 내역**: 지금 어느 단계인지 한눈에 보기, 취소(사유 선택), 다시 주문하기
- **마이페이지**: 닉네임, 즐겨찾기, 최근 주문, **배송지(주소) 관리**
- **회원가입/로그인**: 손님 계정으로 로그인하면 헤더에 표시됨
- **이벤트 페이지**: 지금까지 한 이벤트 전부 보기, 끝난 이벤트는 자동으로 "종료" 표시
- **화면 테마**: 밝은/어두운 모드 전환 스위치, 새로고침해도 유지됨
- **접근성**: 키보드로도 다닐 수 있게 배려

### 관리자가 보는 화면
- **로그인**: 손님 화면과 완전히 분리된 관리자 전용 로그인
- **메뉴 관리**: 카드 형태로 보기, 검색, 여러 개 한번에 품절/판매중/삭제 처리, 사진은 파일 업로드나 링크 둘 중 아무거나
- **카테고리 관리**: 추가/이름 변경/삭제 (쓰는 메뉴 있으면 삭제 못 하게 막음)
- **주문 관리**: 상태별로 보기, 검색, 하나씩 또는 여러 개 한번에 상태 바꾸기
- **공지 관리**: 공지 등록하고 홈 화면에 보여줄 것만 골라서 노출(최대 3개)
- **이벤트 관리**: 등록, 종료 예정일 정해두면 자동으로 종료 처리되거나 직접 종료 체크
- **대시보드**: 전체 메뉴/주문/매출 요약 한눈에 보기

## 4. 겪었던 문제들 (트러블슈팅)

만들면서 실제로 부딪혔던 문제랑 어떻게 해결했는지 정리했어요. 눈으로 코드만 보는 게 아니라 **직접 화면을 띄워 클릭해보면서** 발견한 것들이 많아요.

1. **새로고침하면 주문 상세가 안 뜨는 문제**
   주문 완료 직후엔 잘 보이는데, 새로고침하거나 다른 화면에서 링크 타고 들어가면 안 보이는 현상이 있었어요. 알고 보니 화면을 띄워주는 프로그램이 주소를 정리하는 과정에서 주문 번호 정보를 같이 날려버리고 있었어요. 그 정리 기능을 꺼서 해결했어요.

2. **일부 페이지에서 화면이 아예 깨지는 문제**
   주소가 살짝 다른 형태로 바뀌면서 스타일/스크립트 파일을 엉뚱한 곳에서 찾으려다 깨지는 경우가 있었어요. 파일 경로를 항상 같은 방식으로 고정해서 해결했어요.

3. **메뉴 카드에 마우스를 올려도 반응이 없던 문제**
   카드가 화면에 나타날 때 쓰는 등장 효과가, 끝난 뒤에도 계속 값을 붙잡고 있어서 마우스 올렸을 때 반응이 안 먹히고 있었어요. 등장 효과 설정을 살짝 바꿔서 해결했어요.

4. **배너 사진이 처음 뜰 때 뚝 하고 바로 나타나던 문제**
   원래는 부드럽게 나타나야 하는데 맨 처음 한 번만 바로 나타나는 버그가 있었어요. 화면이 한 번 그려지는 타이밍을 기다렸다가 효과를 주도록 순서를 조정해서 해결했어요.

5. **휴대폰 화면에서 좌우로 스크롤되던 문제**
   좁은 화면에서 홈 화면 일부가 옆으로 밀려서 스크롤되는 버그가 있었어요. 배경에 깔린 장식용 효과가 화면보다 커서 삐져나오고 있던 거였고, 넘치는 부분을 숨기도록 처리해서 해결했어요.

6. **테스트 코드가 아예 실행이 안 되던 문제**
   설정 파일 하나가 잘못 되어있어서 테스트가 전부 실패하고 있었어요. 설정값을 바로잡아서 해결했어요.

7. **사진 올리면 저장 공간이 금방 꽉 차던 문제**
   사진을 원본 그대로 저장하다 보니 몇 장만 올려도 용량이 넘쳐버렸어요. 사진을 자동으로 줄이고 압축해서 저장하도록 바꿨고, 그래도 용량이 넘치면 안내 문구가 뜨게 했어요.

> **어떻게 확인했나**: 그냥 코드만 읽고 넘어간 게 아니라, 실제 브라우저를 띄워서 클릭/입력까지 자동으로 해보는 방식으로 확인했어요. 위 문제들은 전부 이렇게 직접 화면을 조작해보다가 발견한 것들이에요.

## 5. 앞으로 더 하고 싶은 것

- **진짜 데이터베이스로 옮기기**: 지금은 각자 브라우저 안에만 데이터가 저장돼서, 다른 기기나 다른 사람이랑 데이터를 같이 못 봐요. 이걸 실제 서버/데이터베이스로 옮기는 게 제일 먼저 할 일이에요.
- **진짜 결제 연동하기**: 지금은 결제를 "완료됐다고 치는" 수준이라, 실제 카드 결제까지 되게 만들고 싶어요.
- **로그인 더 안전하게 만들기**: 지금은 비밀번호를 그냥 저장해두는 수준이라, 안전하게 암호화하고 다른 방식 로그인(소셜 로그인 등)도 추가하고 싶어요.
- **매출 통계 화면**: 여러 사람 주문이 한곳에 모여야 의미 있는 통계가 나오니까, 데이터베이스 옮긴 다음에 만들고 싶어요.
- **새 주문 실시간 알림**: 관리자가 새 주문 들어온 걸 바로 알 수 있게 알림 기능 추가하고 싶어요.
- **사진 저장 방식 개선**: 지금은 사진을 브라우저 저장소에 직접 넣고 있는데, 별도 저장 공간을 써서 용량 걱정 없게 만들고 싶어요.
- **주문 처리 안정성 높이기**: 주문 저장하는 여러 단계(재고 차감, 주문 기록, 결제 기록)가 중간에 하나라도 실패하면 전부 취소되도록 안전하게 묶고 싶어요.
